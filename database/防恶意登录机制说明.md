# 防止恶意登录机制说明

## 已实现的功能

### 1. 现有组件识别

#### 过滤器 (Filters)
- **VisitCountFilter** - 访问统计过滤器 (`com.hx.filter.VisitCountFilter`)
  - 路径: `/*` (拦截所有请求)
  - 功能: 统计网站访问人数

- **LoginSecurityFilter** - 登录安全过滤器 (`com.hx.filter.LoginSecurityFilter`) 【新增】
  - 路径: `/login` (拦截登录请求)
  - 功能: 防止暴力破解，限制同一IP的登录尝试次数

#### 监听器 (Listeners)
- **AppContextListener** - 应用上下文监听器 (`com.hx.listener.AppContextListener`)
  - 功能: 应用启动时初始化访问计数器，关闭时优雅关闭数据库连接池

#### 连接池 (Connection Pool)
- **HikariCP** - 高性能JDBC连接池 (`com.hx.util.JdbcUtil`)
  - 配置位置: `src/main/resources/db.properties`
  - 功能: 管理数据库连接，提升性能

### 2. 新增防恶意登录机制

#### A. 算术题验证码 (CaptchaUtil)
- **位置**: `com.hx.util.CaptchaUtil`
- **功能**:
  - 生成随机算术题（加法或减法）
  - 数字范围: 1-29，确保计算简单
  - 验证用户答案是否正确

#### B. 登录页面增强 (login.jsp)
- **新增内容**:
  - 算术题显示区域
  - 答案输入框
  - 每次刷新页面生成新题目

#### C. 登录Servlet增强 (LoginServlet)
- **新增验证流程**:
  1. 首先验证算术题答案
  2. 答案错误则记录失败尝试并返回错误
  3. 答案正确才继续验证用户名和密码
  4. 登录成功后清除验证码和尝试记录

#### D. IP锁定机制 (LoginSecurityFilter)
- **安全策略**:
  - 最大尝试次数: 5次
  - 锁定时长: 5分钟
  - 记录每个IP的失败尝试
  - 超过阈值自动锁定IP

## 工作流程

### 正常登录流程
```
用户访问登录页面 
  ↓
生成算术题并存储答案到Session
  ↓
用户输入用户名、密码、算术题答案
  ↓
LoginSecurityFilter检查IP是否被锁定
  ↓
LoginServlet验证算术题答案
  ↓
答案正确 → 验证用户名密码
  ↓
登录成功 → 清除尝试记录，跳转到对应页面
```

### 恶意登录防护流程
```
连续失败尝试
  ↓
每次失败记录IP和尝试次数
  ↓
达到5次失败
  ↓
IP被锁定5分钟
  ↓
锁定期间所有登录请求被拒绝
  ↓
5分钟后自动解锁
```

## 测试方法

### 1. 测试算术题验证
1. 访问登录页面: http://localhost:8080/你的项目名/login.jsp
2. 查看页面上显示的算术题（如: 7 + 5 = ?）
3. 故意输入错误答案
4. 应该看到错误提示: "算术题答案错误，请重新计算！"

### 2. 测试正常登录
1. 输入正确的算术题答案
2. 使用测试账号登录:
   - 管理员: admin / admin123
   - 普通用户: student1 / 123456
3. 应该成功登录

### 3. 测试IP锁定
1. 连续5次输入错误的密码或算术题答案
2. 第5次失败后应该看到: "登录尝试次数过多，账户已被锁定5分钟，请稍后再试！"
3. 在5分钟内无法再尝试登录
4. 5分钟后自动解锁

## 技术特点

### 安全性
- ✅ 算术题每次刷新都变化
- ✅ 答案存储在Session中，防止前端篡改
- ✅ 使用后立即清除，防止重放攻击
- ✅ IP级别的安全控制
- ✅ 自动锁定机制

### 用户体验
- ✅ 算术题简单易懂（10以内的加减法）
- ✅ 界面友好，有清晰的提示
- ✅ 锁定时间合理（5分钟）
- ✅ 自动解锁，无需人工干预

### 性能优化
- ✅ 使用ConcurrentHashMap保证线程安全
- ✅ 自动清理过期的锁定记录
- ✅ 不影响正常用户的登录速度

## 配置参数调整

如需调整安全策略，可修改 `LoginSecurityFilter.java`:

```java
// 最大登录尝试次数（默认5次）
private static final int MAX_ATTEMPTS = 5;

// 锁定时长（默认5分钟）
private static final long LOCK_TIME = 5 * 60 * 1000;
```

## 注意事项

1. **开发环境测试**: 如果需要频繁测试，可以临时增加MAX_ATTEMPTS的值
2. **生产环境**: 建议将锁定信息持久化到数据库，防止服务器重启后丢失
3. **代理环境**: 已支持获取真实IP（X-Forwarded-For等头部）
4. **Session超时**: 确保web.xml中配置的session超时时间合理
