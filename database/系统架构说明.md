# 在线考试系统 - 架构与实现流程说明

## 项目概述

这是一个基于 Java Web 的在线考试系统，采用 MVC 三层架构（Model-View-Controller），使用 JSP + Servlet + JDBC 技术栈实现。

---

## 一、项目结构说明

```
test1/
├── src/main/
│   ├── java/com/hx/          # Java 源代码
│   │   ├── dao/              # 数据访问层 (Data Access Object)
│   │   ├── entity/           # 实体层 (Entity/Model)
│   │   ├── servlet/          # 控制层 (Controller)
│   │   └── util/             # 工具类
│   └── webapp/               # Web 资源
│       ├── WEB-INF/          # 配置文件
│       ├── admin/            # 管理员页面
│       ├── error/            # 错误页面
│       └── *.jsp             # 用户页面
├── database/                 # 数据库脚本
└── pom.xml                   # Maven 配置
```

## 二、各文件详细说明

### 1. 实体层 (Entity) - Model

#### `Users.java`
**作用**：用户信息的数据模型  
**功能**：
- 封装用户相关属性：userId, userName, password, sex, email, role
- 提供 getter/setter 方法访问属性
- 作为数据传输对象 (DTO)，在各层之间传递用户数据

**使用场景**：
- DAO 层查询结果封装
- Servlet 接收表单数据
- JSP 页面显示用户信息

#### `Question.java`
**作用**：试题信息的数据模型  
**功能**：
- 封装试题属性：questionId, title, optionA-D, answer
- 提供 getter/setter 方法
- 用于试题数据的传输和存储

---

### 2. 数据访问层 (DAO) - Data Access Layer

#### `UsersDAO.java`
**作用**：用户数据的数据库操作封装  
**功能**：
- `login()` - 用户登录验证
- `register()` - 用户注册
- `findById()` - 根据 ID 查询用户
- `findAll()` - 查询所有用户
- `update()` - 更新用户信息
- `delete()` - 删除用户

**技术要点**：
- 使用 JdbcUtil 获取数据库连接
- PreparedStatement 防止 SQL 注入
- ResultSet 处理查询结果
- 返回 Users 对象或列表

#### `QuestionDAO.java`
**作用**：试题数据的数据库操作封装  
**功能**：
- 查询所有试题
- 随机抽取试题组卷
- 查询特定试题
- 添加试题
- 删除试题

---

### 3. 控制层 (Servlet) - Controller

#### 用户功能模块

##### `LoginServlet.java`
**作用**：处理用户登录请求  
**流程**：
1. 接收用户名和密码参数
2. 调用 UsersDAO.login() 验证
3. 验证成功：创建 Session，跳转首页
4. 验证失败：返回错误信息

##### `RegisterServlet.java`
**作用**：处理用户注册请求  
**流程**：
1. 接收注册表单数据
2. 创建 Users 对象
3. 调用 UsersDAO.register() 插入数据库
4. 返回注册结果

##### `LogoutServlet.java`
**作用**：处理用户登出  
**流程**：
1. 使 Session 失效
2. 重定向到登录页

##### `StartExamServlet.java`
**作用**：开始考试  
**流程**：
1. 从数据库随机抽取试题
2. 将试题存入 Session
3. 跳转到 exam.jsp

##### `SubmitExamServlet.java`
**作用**：提交试卷并评分  
**流程**：
1. 接收用户答案
2. 与正确答案对比
3. 计算得分
4. 保存成绩
5. 跳转到成绩页面

##### `UpdateUserServlet.java`
**作用**：更新用户信息  
**流程**：
1. 接收修改后的用户信息
2. 调用 UsersDAO.update()
3. 返回更新结果

#### 管理员功能模块

##### `admin/UserManageServlet.java`
**作用**：用户管理  
**功能**：
- 查询所有用户
- 删除指定用户
- 支持批量操作

##### `admin/QuestionManageServlet.java`
**作用**：试题管理  
**功能**：
- 查询所有试题
- 分页显示
- 搜索过滤

##### `admin/AddQuestionServlet.java`
**作用**：添加试题  
**流程**：
1. 接收试题表单数据
2. 创建 Question 对象
3. 调用 QuestionDAO 插入数据库
4. 返回添加结果

##### `admin/DeleteQuestionServlet.java`
**作用**：删除试题  
**流程**：
1. 接收试题 ID
2. 调用 QuestionDAO.delete()
3. 返回删除结果

##### `admin/DeleteUserServlet.java`
**作用**：删除用户  
**流程**：
1. 接收用户 ID
2. 调用 UsersDAO.delete()
3. 返回删除结果

## 三、核心业务流程模拟

### 流程 1：用户登录流程

```
1. 用户访问 index.jsp
   ↓
2. 点击"登录"按钮，跳转到 login.jsp
   ↓
3. 填写用户名和密码，提交表单
   ↓
4. LoginServlet 接收请求
   ├─ 获取参数：userName, password
   ├─ 调用 UsersDAO.login(userName, password)
   │   ├─ JdbcUtil.getConnection() 获取数据库连接
   │   ├─ 执行 SQL: SELECT * FROM Users WHERE userName=? AND password=?
   │   ├─ 查询结果封装为 Users 对象
   │   └─ 返回 user 对象
   ├─ 如果 user != null
   │   ├─ session.setAttribute("currentUser", user)
   │   └─ response.sendRedirect("index.jsp")
   └─ 如果 user == null
       └─ request.setAttribute("error", "用户名或密码错误")
           request.getRequestDispatcher("login.jsp").forward(...)
   ↓
5. 登录成功，返回首页显示用户信息
```

---

### 流程 2：在线考试流程

```
1. 已登录用户点击"开始考试"
   ↓
2. StartExamServlet 接收请求
   ├─ 调用 QuestionDAO.findAll() 查询所有试题
   ├─ 随机抽取 10 道试题
   ├─ 将试题列表存入 Session
   │   session.setAttribute("examQuestions", questions)
   └─ 转发到 exam.jsp
   ↓
3. exam.jsp 显示试题
   ├─ 从 Session 获取试题列表
   ├─ 遍历试题，渲染题目和选项
   ├─ 提供单选框供用户选择答案
   └─ 显示倒计时
   ↓
4. 用户答题并提交
   ↓
5. SubmitExamServlet 处理提交
   ├─ 获取用户答案：request.getParameterValues("answers")
   ├─ 从 Session 获取试题和正确答案
   ├─ 逐题对比，计算得分
   │   int score = correctCount * 10;
   ├─ 将成绩存入 Session
   │   session.setAttribute("score", score);
   └─ 重定向到 score.jsp
   ↓
6. score.jsp 显示成绩
   ├─ 从 Session 获取分数
   ├─ 显示答题情况
   └─ 提供"重新考试"选项
```

---

### 流程 3：用户注册流程

```
1. 新用户访问 register.jsp
   ↓
2. 填写注册信息
   ├─ 用户名
   ├─ 密码
   ├─ 确认密码
   ├─ 性别
   └─ 邮箱
   ↓
3. 提交表单到 RegisterServlet
   ↓
4. RegisterServlet 处理
   ├─ 获取表单参数
   ├─ 验证数据（密码一致性、邮箱格式等）
   ├─ 创建 Users 对象
   │   user.setUserName(...)
   │   user.setPassword(...)
   │   user.setSex(...)
   │   user.setEmail(...)
   │   user.setRole("user")
   ├─ 调用 UsersDAO.register(user)
   │   ├─ SQL: INSERT INTO Users (userName, password, sex, email, role) VALUES (?, ?, ?, ?, ?)
   │   ├─ 设置参数
   │   ├─ executeUpdate() 执行插入
   │   └─ 返回 success/failure
   └─ 根据结果跳转
       ├─ 成功：redirect to login.jsp
       └─ 失败：forward to register.jsp with error message
   ↓
5. 注册成功，跳转到登录页面
```

### 流程 4：管理员管理试题流程

```
1. 管理员登录系统
   ↓
2. 访问 admin/questionList.jsp
   ↓
3. QuestionManageServlet 加载试题
```

### 流程 5：数据库查询完整流程

```
以查询所有用户为例：

1. UserManageServlet 需要获取用户列表
   ↓
2. 调用 UsersDAO.findAll()
   ↓
3. UsersDAO.findAll() 执行
   ├─ List<Users> userList = new ArrayList<>()
   ├─ Connection conn = JdbcUtil.getConnection()
   ├─ String sql = "SELECT * FROM Users ORDER BY userId"
   ├─ PreparedStatement ps = conn.prepareStatement(sql)
   ├─ ResultSet rs = ps.executeQuery()
   ├─ while(rs.next()) {
   │     Users user = new Users()
   │     user.setUserId(rs.getInt("userId"))      // columnLabel: userId
   │     user.setUserName(rs.getString("userName")) // columnLabel: userName
   │     user.setPassword(rs.getString("password")) // columnLabel: password
   │     user.setSex(rs.getString("sex"))           // columnLabel: sex
   │     user.setEmail(rs.getString("email"))       // columnLabel: email
   │     user.setRole(rs.getString("role"))         // columnLabel: role
   │     userList.add(user)
   │   }
   ├─ JdbcUtil.closeAll(rs, ps, conn)
   └─ return userList
   ↓
4. Servlet 获取 userList
   ↓
5. request.setAttribute("userList", userList)
   ↓
6. forward 到 userList.jsp
   ↓
7. JSP 使用 JSTL 遍历显示
   <c:forEach items="${userList}" var="user">
       ${user.userName} | ${user.email} | ${user.role}
   </c:forEach>
```

## 四、关键技术点说明

### 1. ColumnLabel（列标签）

**定义**：ResultSet 中列的标识符，用于从查询结果中获取特定列的值

**示例**：
```java
rs.getInt("userId")      // "userId" 是 columnLabel
rs.getString("userName") // "userName" 是 columnLabel
```

**匹配规则**：
- 必须与数据库表字段名一致
- 来自 SQL 查询的 SELECT 子句
- 在 UsersDAO 中使用的是 Users 表的字段名

### 2. MVC 三层架构

**Model（模型）**：
- Users.java, Question.java
- 负责数据封装和传输

**View（视图）**：
- 所有 JSP 页面
- 负责数据显示

**Controller（控制器）**：
- 所有 Servlet
- 负责请求处理和流程控制

### 3. DAO 模式

**优势**：
- 分离业务逻辑和数据访问
- 提高代码可维护性
- 便于单元测试

### 4. Session 管理

**使用场景**：
- 用户登录状态保持
- 考试过程中试题暂存
- 成绩临时存储

**关键代码**：
```java
// 设置 Session
session.setAttribute("currentUser", user);

// 获取 Session
Users user = (Users) session.getAttribute("currentUser");

// 销毁 Session
session.invalidate();
```
